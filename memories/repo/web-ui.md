- Table blind badges are gated by `TableCanvas.IsBlindBasedGame`; if a blind-based variant is missing there, `TableSeat` will never receive `IsSmallBlind` or `IsBigBlind`.
- Bob Barker showdown rendering can receive either `4 hole + shared board` or `5 player cards + 5 board`; normalize via the separate showcase card before computing/rendering the best five.
- Bob Barker showcase timeout selection lives in `DrawPanel`; preserve a valid manual selection, otherwise auto-pick the lowest singleton rank with Ace low by default, falling back to breaking the smallest duplicate group.
- Pair Pressure wild highlighting in web UI should use active `WildCardRules.WildRanks` from table state rather than per-card `IsWild` flags or showdown `WildCardIndexes`, because when the tracked wild ranks rotate all cards of the active rank must update together.
- During Pair Pressure deal animation, derive active wild ranks from currently revealed face-up seat cards only; do not let server wild ranks or per-card `IsWild` flags turn cards green before the matching face-up card has actually been animated in.
- Pair Pressure dashboard odds must use the ordered stream of currently visible face-up cards and treat those exposed opponent cards as unavailable in the simulation deck; plain stud odds are not sufficient because the active wild ranks rotate as new pairs appear.
- Proper Pair Pressure odds also need non-hero hidden dealt cards removed from the deck and future street cards simulated for every live seat in stud rotation order; otherwise current wild ranks are frozen and impossible outcomes leak into the dashboard.
- Stud-style ordering is now centralized in `StudOrderHelper` under the poker library; API table-state card ordering and web dashboard face-up reconstruction should use that helper instead of reimplementing phase/deal-order logic locally.
- The animated Pair Pressure wild-card panel should derive active ranks from the same ordered visible face-up stream as the dashboard odds path and run that stream through `PairPressureWildCardRules`; do not show every visible paired rank once a third distinct pair has rotated the oldest one out.
